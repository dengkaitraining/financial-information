# 請使用 python GNews 套件建構一「Google 新聞爬蟲」單元，詳細規格如<spec>。
<spec>
 1. 「Google 新聞爬蟲」單元以 python 物件導向的規範編寫，封裝為 class。
 2. GNews 模組初始參數設定：
    | 參數 | 設定值 | 用途說明 |
    | --- | --- | --- |
    | language | 'zh-TW' | 語系地區 |
    | country | 'TW' | 地區 |
    | period | '4h' | 過去 4 小時的新聞 |
    | max_results | 100 | 單次抓取量 |
 3. 新聞資料欄位
    | 欄位 | 用途說明 |
    | --- | --- |
    | **('title')** | 新聞標題 |
    | **('published date')** | 發布時間 |
    | **('publisher')** | 新聞來源 |
    | **('url'))** | 文章連結 |
    | **('description')** | 說明摘要 |
 4. 依據「2.」、「3.」，分別建立「參數設定表」單與「新聞資料表單」，並生成 MariaDB 12.3 的 SQL create table 指令檔。
 5. 依據「2.」的資訊寫入「參數設定表單」，並使用此參數作為 GNews 物件的初始值。
 6. 透過「Google 新聞爬蟲」的方式，抓取「3.」資訊並經使用者確認後，再將資料寫入「新聞資料表單」並確認「新聞資料沒有重複」(MariaDB 12.3資料庫)。
 7. 最後產生 CLI 的測試界面，完整模組與測試程式分開成不同檔案，以確認 class 內各項功能正常運行。
</spec>
