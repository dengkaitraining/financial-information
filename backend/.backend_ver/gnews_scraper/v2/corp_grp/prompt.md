# 請使用 python FinMind 與 yfinance 套件建構一抓取「公司詳細資訊」單元，詳細規格如<spec>。
<spec>
 1. 「公司詳細資訊」單元以 python 物件導向的規範編寫，封裝為 class。
 2. 股票分析常用資料欄位 (Taxonomy)
    | 類別 | 常用關鍵欄位 | 分析用途 |
    | --- | --- | --- |
    | **公司基本面** | 產業類別 (`industry_category`)、資本額 (`capital`)、成立/上市日期 (`date`) | 產業分類比對、規模大小篩選 |
    | **經營營運面** | 月營收 (`revenue`)、營收年增率 (YoY)、月增率 (MoM) | 評估公司短期營運成長動能 |
    | **財務獲利面** | 每股盈餘 (EPS)、毛利率、營業利益率、負債比率、ROE/ROA | 評估公司長期獲利能力與財務體質 |
    | **估值與籌碼** | 本益比 (P/E)、股價淨值比 (P/B)、殖利率 (`yield_rate`)、三大法人買賣超 | 評估目前股價是否偏離價值、籌碼動向 |
 3. 依據「公司基本面」、「經營營運面」、「財務獲利面」、「估值與籌碼」類別，分別建立不同的表單與常用關鍵欄位，並生成 MariaDB 12.3 的 SQL create table 指令檔
 4. 透過搜尋公司資訊的方式，抓取「2.」資訊並經使用者確認後，再分別將資料寫入「公司基本面」、「經營營運面」、「財務獲利面」、「估值與籌碼」等表單資訊(MariaDB 12.3資料庫)。
 5. 最後產生 CLI 的測試界面，完整模組與測試程式分開成不同檔案，以確認 class 內各項功能正常運行。
</spec>
